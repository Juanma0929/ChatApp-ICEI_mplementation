En los mismos grupos de la Tarea 1.

Contexto
En la Tarea 1, implementaron un sistema de chat basado en comunicación por sockets TCP, en el cual los clientes en Java podían conectarse a un servidor y realizar acciones como enviar mensajes, notas de voz y llamadas, además de crear grupos y mantener un historial de mensajes.

En la Tarea 2, debieron reutilizar su proyecto anterior y adaptarlo para que el cliente funcione bajo el protocolo HTTP. El objetivo fue comenzar la transición hacia un entorno web, donde el cliente se implementará en HTML, CSS y JavaScript, mientras que el servidor principal seguirá siendo el backend en Java.

Para este Proyecto final de curso deberán migrar el proyecto a una implementación con RPC (utilizado el middleware de ZeroC Ice) en este caso la conexión entre los clientes y el servidor será por medio de Ice, utilizando el frontend de la tarea 2 y el backend ya implementado. Los servicios que ya conectaron usando HTTP no es necesario que los migren a ICE sin embargo lo pueden hacer. Deben usar ws (WebSockets) de ICE para reflejar un comportamiento en tiempo real de todos los servicios.



Requerimientos funcionales
El nuevo cliente deberá permitir las siguientes acciones a través del RPC:

Crear grupos de chat.

Enviar mensajes de texto a un usuario específico o a un grupo. debe actualizarse en tiempo real

Visualizar el historial de mensajes (texto y audios) asociados al usuario o grupo.

Envío de notas de voz y/o llamadas desde el navegador, tenga en cuenta que para esta parte se deben utilizar WebSockets.

Nota: Para el punto 4, pueden tomar de referencia el ejemplo de código: https://github.com/AlejandroMu/compu-internet-1/tree/master/audio_rep en donde se implementa, gravación de audio desde el servidor (backend en java) y reproducción en el cliente (JavaScript) ustedes deberán implementar la grabación de audio desde el cliente  

Requerimientos técnicos
El cliente debe estar desarrollado en HTML, CSS y JavaScript vanilla y usar webpack para su empaquetado.

La comunicación con el backend Java debe realizarse mediante la conexión por RPC (usando ZeroC Ice) o HTTP usando express

Se debe conservar la lógica del servidor y las estructuras de datos de los proyectos anterior.
